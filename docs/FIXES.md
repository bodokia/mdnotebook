# Протокол изменений mdnotebook

Сформирован по результатам UX-аудита. Изменения сгруппированы по приоритету.

---

## 🔴 Критические (блокируют мобайл)

### FIX-01 — Hover-only действия недоступны на iPhone
**Файл:** `src/components/Sidebar.tsx`  
**Проблема:** Кнопки Rename (`✎`) и Delete (`✕`) появляются только при `onMouseEnter`. На тачскрине hover не срабатывает — на iPhone нельзя ни переименовать, ни удалить заметку.  
**Решение:** Заменить hover-логику на long-press для мобайла. На десктопе оставить hover. Альтернатива — контекстное меню по правой кнопке (desktop) и long-press (mobile), либо всегда показывать кнопки на активном файле.

---

### FIX-02 — Нативный `prompt()` / `confirm()` вместо UI-диалогов
**Файл:** `src/components/Sidebar.tsx`  
**Проблема:** Создание заметки через `prompt('Note name:')`, удаление через `confirm(...)`. На iOS нативные диалоги выглядят как системные алерты, не соответствуют стилю приложения. В ряде конфигураций `prompt()` блокируется браузером.  
**Решение:** Встроенный inline-инпут для создания/переименования прямо в сайдбаре. Маленький кастомный confirm-диалог для удаления.

---

### FIX-03 — Dark mode: код в блоках нечитаем
**Файл:** `src/components/Preview.tsx`  
**Проблема:** Всегда импортируется `highlight.js/styles/github.css` (светлая тема). В тёмном режиме переопределяется только фон (`--surface`), но цвета токенов остаются для светлого фона — тёмный текст на тёмном фоне.  
**Решение:** Динамически подключать `github-dark.css` в dark mode. Либо условный import через CSS (`:root` / `.dark` переменные). Простейший вариант — два `<link>` с `media`, или замена через динамический `import()`.

---

### FIX-04 — Mobile toolbar: тапзоны меньше 44px
**Файл:** `src/App.tsx`  
**Проблема:** Кнопки Edit / Preview в мобильном тулбаре (`px-2 py-1 text-xs`) имеют высоту ~28px — ниже порога HIG (44px).  
**Решение:** Увеличить `minHeight: 44` и `minWidth: 44` для всех кнопок тулбара на мобайле.

## 🟠 Важные

### FIX-05 — Удаление без возможности отмены
**Файл:** `src/components/Sidebar.tsx`, `src/App.tsx`  
**Проблема:** Файл удаляется безвозвратно, нет undo, нет корзины.  
**Решение:** Минимально — toast-уведомление с кнопкой «Отменить» (5 секунд, реальное удаление откладывается). Либо мягкое удаление в `.trash/` папку.

---

### FIX-06 — Нет обратной связи при ошибке сохранения
**Файл:** `src/App.tsx`  
**Проблема:** Если автосохранение упало (нет сети, диск заполнен) — `●` исчезает, пользователь думает что сохранилось. Данные потеряны молча.  
**Решение:** Оборачивать `saveFile()` в try/catch, показывать ошибку в toolbar (красный индикатор, текст «Ошибка сохранения»). Повторная попытка при следующем изменении.

---

### FIX-07 — Нет сброса позиции скролла при смене файла
**Файл:** `src/components/Editor.tsx`, `src/App.tsx`  
**Проблема:** При открытии нового файла редактор и превью остаются на той же позиции скролла — пользователь попадает в середину документа.  
**Решение:** При смене `selectedPath` вызывать `scrollTop = 0` на контейнерах редактора и превью.

## 🟡 Нужные улучшения

### FIX-08 — Нет поиска по заметкам
**Проблема:** При большом количестве заметок найти нужную можно только прокруткой дерева.  
**Решение:** Поле поиска в шапке сайдбара. Фильтрует файловое дерево по имени файла (минимум). Опционально — поиск по содержимому через backend (`GET /api/search?q=...`).

---

### FIX-09 — Нет переключателя шрифта в редакторе
**Файл:** `src/components/Editor.tsx`  
**Проблема:** По плану ([PLAN.md](PLAN.md) Фаза 5) должен быть переключатель `mono` / `serif` в тулбаре редактора. Не реализован — редактор всегда показывает JetBrains Mono. Пользователь пишет прозу в моноширинном шрифте.  
**Решение:** Кнопка-переключатель в тулбаре (или в самом редакторе). Сохранять выбор в `localStorage`.

---

### FIX-10 — Нельзя создать заметку в конкретной папке
**Файл:** `src/components/Sidebar.tsx`  
**Проблема:** `+` в шапке сайдбара всегда создаёт файл в корне `NOTES_DIR`. Нет способа создать заметку внутри существующей папки.  
**Решение:** Кнопка `+` рядом с папкой при hover/long-press, создаёт файл внутри этой папки.

---

### FIX-11 — Нельзя переименовать или создать папку
**Файл:** `src/components/Sidebar.tsx`  
**Проблема:** Кнопки Rename/Delete отрендерены только для `node.type === 'file'`. Папки нельзя переименовать. Нет способа создать папку.  
**Решение:** Добавить rename-кнопку для папок. Кнопку «+ Папка» в шапке или рядом с существующими папками.

---

### FIX-12 — Нет индикатора успешного сохранения
**Файл:** `src/App.tsx`  
**Проблема:** Индикатор `●` исчезает когда `content === savedContent`, но нет подтверждения что сохранение прошло успешно. Пользователь не уверен в состоянии файла.  
**Решение:** Кратковременный «Сохранено» текст или checkmark (исчезает через 2 сек) после успешного `saveFile()`.

---

### FIX-13 — Enter в Markdown-списке не продолжает маркер
**Файл:** `src/components/Editor.tsx`  
**Проблема:** Нажатие Enter внутри строки `- item` не добавляет `- ` на следующей строке. Стандартное поведение Markdown-редакторов.  
**Решение:** В `handleKeyDown` при Enter проверять текущую строку на начало с `- `, `* `, `1. ` и продолжать паттерн.

---

### FIX-16 — WYSIWYG-редактор (редактирование в визуальном виде)
**Файл:** `src/components/Editor.tsx`, `src/App.tsx` — замена архитектуры редактора  
**Проблема:** Текущий редактор — сырой `<textarea>` с Markdown-символами. Пользователь вынужден держать в голове синтаксис (`**`, `##`, `- `) и переключаться в Preview чтобы увидеть результат. Цель — редактировать текст в том виде, в котором он выглядит: заголовок сразу выглядит заголовком, жирный — жирным, список — списком. Символы Markdown не видны, но хранятся в файле.  

**Аналоги:** Typora, Notion, Obsidian Live Preview, приложение Minimal.  

**Решение — заменить `<textarea>` на TipTap:**
- [TipTap](https://tiptap.dev) — ProseMirror-based WYSIWYG-редактор с расширением `@tiptap/extension-markdown`
- Пользователь видит и редактирует отформатированный текст; файл на диске остаётся чистым `.md`
- При загрузке файла: Markdown → TipTap JSON (через `@tiptap/extension-markdown`)
- При сохранении: TipTap JSON → Markdown → файл
- Шрифты сохраняются: Lora для текста, JetBrains Mono для `code`-блоков
- Поддерживаемые элементы: заголовки H1–H3, жирный, курсив, inline-код, блоки кода, списки (маркированные и нумерованные), task-листы (`- [ ]`), горизонтальные линии, ссылки
- Режим Split становится: WYSIWYG-редактор слева | полный Preview справа (для финального контроля)
- Старый режим «Edit» (сырой Markdown) можно оставить как переключаемый «Raw» режим для опытных пользователей

**Пакеты:**
```
@tiptap/react
@tiptap/starter-kit
@tiptap/extension-markdown
@tiptap/extension-task-list
@tiptap/extension-task-item
@tiptap/extension-link
@tiptap/extension-code-block-lowlight
lowlight
```

**Влияние на архитектуру:** Editor.tsx переписывается полностью. Preview.tsx остаётся без изменений. Автосохранение, dirty-check и Cmd+S не меняются — они работают со строкой Markdown, которую теперь отдаёт TipTap.

## 📝 Второстепенное (polish)

### FIX-14 — NOTES_DIR по умолчанию = папка проекта
**Файл:** `server.ts`  
**Проблема:** При запуске без `NOTES_DIR` пользователь видит технические файлы (`CLAUDE.md`, `docs/ARCHITECTURE.md`) вместо личных заметок.  
**Решение:** Изменить default на `~/Documents/Notes` или `~/Notes`. Либо показывать экран первого запуска с предложением указать папку.

**Проблема:** «Select a note or create a new one», «No notes yet», «Loading…», «Note name:» — пользователь ожидает русский интерфейс.  
**Решение:** Перевести все строки. Либо добавить минимальный i18n (объект с ключами).
